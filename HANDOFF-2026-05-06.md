## Next Session: Three Remaining Tasks

### 1. Deduplication — Don't re-enrich already-enriched records

**Problem:** Every weekly run fires Clay for ALL signals with a developer, even if that developer was enriched last week. Wastes Clay credits, pollutes logs.

**Game signals fix needed:**
- Before Clay push, check `game_signals` table: `developer_domain IS NOT NULL AND developer = ?` → skip if already enriched recently (30-day window)
- Filter `enrichable` to only signals where `developer_domain` is null or enrichment fields are blank

```typescript
// In pushToClay(), before creating tokens:
const needsEnrich = enrichable.filter((s) => !s.developer_domain);
// or query Supabase for already-enriched developers
```

**PH fix needed:**
- Check `product_launches` table before Clay stage: if `employee_count IS NOT NULL` for that `source_url`, skip enrichment
- Or simpler: if `maker_website` is already enriched (any enrichment field set), skip

**Cross-run dedup on source_url already works** (Supabase `on_conflict=source_url` + `seenRecently()` in game signals). PH uses `on_conflict=source_url` too. This is solid — same article won't produce two rows.

**Gap:** `seenRecently()` deduplicates the *signal record*, but Clay still fires for every new signal even if the *developer* was enriched before. Need a developer-level enrichment check.

### 3. Source-level dedup across runs

**Current state:**
- Game signals: `seenRecently()` checks by `game_title` (announcements) or `developer + funding_amount` (fundings) — 30-day lookback. `on_conflict=source_url` handles exact URL duplication.
- PH: `on_conflict=source_url` only. No cross-run entity dedup.

**Gaps:**
- Same studio funding covered by two different articles → two rows in game_signals (different source_urls). `seenRecently()` catches this but only if `developer + funding_amount` matches exactly.
- PH: same product launched on two consecutive days (re-featured) → two rows. Need entity dedup by `company_name + discovered_date` or similar.
- News pipeline: no cross-run dedup at all beyond `on_conflict=source_url`.

**Fixes:**
- Game signals: `seenRecently()` already handles most of this. Consider tightening: check by `developer` alone (not requiring matching `funding_amount`) for fundings.
- PH: add `seenRecently()` equivalent — check `product_launches` table for same `product_name` within last 7 days before upsert.
- News: add entity dedup similar to PH — same `company_name + product_name` within 24 hours = skip.
